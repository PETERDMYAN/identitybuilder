# Identity Compound ◮

A personal mobile app for compounding the three accounts of a life — **health, knowledge, relationships** — built on three mental models:

1. **Identity over outcomes.** You don't "want to run"; you *are someone who trains*. Every kept commitment is a **vote of evidence** for that identity (à la *Atomic Habits*).
2. **Inversion.** Alongside each 5-year vision you write the **anti-vision** — who five years of drift would make you. The app keeps that cliff quietly in view (à la Munger: "Invert, always invert").
3. **Compounding.** 1% better daily is ×37.8 in a year; 1% worse is ×0.03. The Growth tab draws both curves and threads **your actual trajectory** between them, week by week.

The operating rhythm: **set the day → keep the day → reflect nightly → review weekly.** No onboarding, no weekly planning ceremony: every morning you write what you *will* do and what you *won’t*, and check them off.

| Tab | What it does |
| --- | --- |
| **Today** | Set today’s "I will" and "I won’t" lists (optionally tagged to an account), check them off, "I commit to today", evening "Close the day" (alignment 1–5 + one honest line), gentle inversion nudge |
| **Week** | The week's commitments as a 7-day dot grid, kept/target progress, weekly reflection, recent weeks |
| **Growth** | Compound score per account and for Life, the two-curves wedge chart, streak, total votes, pace → 1 year, evidence log |
| **Compass** | Identity statements, 5-year visions, anti-visions, life goals — all editable |

Data lives in **your own AWS account**: Cognito for login, AppSync + DynamoDB for storage, every record owner-scoped to your user. Nothing is shared with anyone.

## Setup (once)

Prereqs: Node 20+, AWS CLI configured (`aws configure`) with an IAM user that can deploy CloudFormation/Cognito/AppSync/DynamoDB.

```bash
npm install
npx ampx sandbox --once   # provisions the backend, writes amplify_outputs.json
npx expo start            # scan the QR with Expo Go on your phone
```

Sign in with Apple — one tap, no passwords, no wizard. Your three accounts (Health, Knowledge, Relationships) are created silently; write the identity statements, visions and inversions in Compass whenever you’re ready.

Keep the real `amplify_outputs.json` committed — it's client-side config (pool ids, endpoint), not secrets, and builds need it.

## Notifications

Morning-commit, evening-reflect, and weekly-plan reminders are scheduled locally (no server). **Expo Go has limited notification support** — they work fully in a real build (TestFlight or `npx expo run:ios`).

## Ship it to TestFlight

Everything is pre-configured: bundle ID `com.peterdmyan.identitybuilder`, icons, export-compliance flag, and the backend config ships inside the app bundle via `amplify_outputs.json` — no build-time env vars needed.

**Local build (this Mac has Xcode):**

```bash
npx expo prebuild -p ios
xcodebuild -workspace ios/IdentityCompound.xcworkspace -scheme IdentityCompound \
  -configuration Release -destination 'generic/platform=iOS' \
  -archivePath build/IdentityCompound.xcarchive archive \
  -allowProvisioningUpdates
xcodebuild -exportArchive -archivePath build/IdentityCompound.xcarchive \
  -exportOptionsPlist ExportOptions.plist -exportPath build \
  -allowProvisioningUpdates
```

`ExportOptions.plist` (in the repo root) uses `method: app-store-connect` with `destination: upload`, so the export step signs *and* uploads to App Store Connect in one go. Then: App Store Connect → TestFlight → add yourself to an internal-tester group → install via the TestFlight app.

**Cloud build (EAS)** also works if you prefer: `eas login && eas init && eas build -p ios --profile production && eas submit -p ios --latest` (needs a free Expo account; Apple Developer Program required either way).

## Stack

Expo SDK 57 (React Native + TypeScript, expo-router) · AWS Amplify Gen 2 (Cognito auth, AppSync GraphQL, DynamoDB with per-owner authorization) · TanStack Query · react-native-svg for the compounding chart · Fraunces & Inter via expo-google-fonts.

```
amplify/          backend definition (TypeScript-first): auth + data schema
src/
  app/            screens (expo-router): tabs, sign-in, reflect-weekly, settings
  components/     UI kit, CompoundChart (the wedge), MementoBanner
  lib/            amplify client, auth, queries (AppSync), compounding math, dates, notifications
amplify_outputs.json   client config generated by `npx ampx sandbox` — committed on purpose
```

## The math

A fully kept week multiplies each account's score by 1.01⁷ ≈ ×1.072, a fully broken one by 0.99⁷ ≈ ×0.932, partial weeks interpolate, and unplanned weeks hold flat (rest isn't decay — only broken commitments are). The chart is drawn in log space, where the two 1%-curves become straight lines: a wedge, with your line inside it.
